# 播放问题排查指南

## 当前状态

播放器界面显示：
- ✅ 歌曲名称显示正常
- ✅ 时长显示正常 (00:00 / 02:39)
- ✅ 音量显示正常 (80%)
- ✅ 播放模式显示正常 (order)
- ❌ 没有声音输出
- ❌ 进度条没有移动

## 可能的原因

### 1. API Token 未正确加载

检查方法：
```bash
# 查看token文件是否存在
ls -la ~/.pan-player/token.json

# 查看token内容
cat ~/.pan-player/token.json
```

### 2. 下载链接获取失败

运行程序时查看调试输出：
```
DEBUG: 开始加载歌曲: xxx.mp3
DEBUG: 加载歌曲失败: xxx错误
```

### 3. 解码器问题

检查解码器实现：
- MP3: 使用 `github.com/faiface/beep/mp3`
- WAV: 使用 `github.com/youpy/go-wav`
- FLAC: 使用 `github.com/faiface/beep/flac`
- OGG: 使用 `github.com/faiface/beep/vorbis`

### 4. Speaker 初始化问题

检查 speaker 是否正确初始化。

## 解决方案

### 方案1：检查 Token

确保已登录并保存了有效的token：

```bash
# 如果token文件存在，删除重新登录
rm ~/.pan-player/token.json

# 重新运行程序，扫描二维码登录
./pan-player
```

### 方案2：检查依赖

确保所有音频解码库已正确安装：

```bash
# 查看go.mod中的依赖
cat go.mod | grep beep
cat go.mod | grep wav

# 重新下载依赖
go mod download
go mod tidy
```

### 方案3：测试音频播放

使用测试文件验证播放器：

```bash
# 运行音频测试
go run test_audio.go /path/to/test.mp3
```

### 方案4：添加详细日志

我已经在代码中添加了调试信息，运行程序时会输出：
- 加载歌曲的文件名
- 加载成功或失败的信息
- 具体的错误原因

查看这些输出来定位问题。

## 常见问题

### Q1: 编译成功但没声音

**检查步骤**：
1. 确认系统音量未静音
2. 查看终端调试输出
3. 检查 token.json 是否有效
4. 尝试播放本地测试文件

### Q2: 提示"加载歌曲失败"

**可能原因**：
- Token 过期（有效期30天）
- 网络连接问题
- 文件格式不支持
- 百度网盘API限制

**解决方法**：
```bash
# 重新登录
rm ~/.pan-player/token.json
./pan-player
```

### Q3: 播放器卡住不动

**可能原因**：
- 异步加载未完成
- Speaker未初始化
- 文件下载卡住

**调试方法**：
查看程序的标准输出，会有详细的调试信息。
