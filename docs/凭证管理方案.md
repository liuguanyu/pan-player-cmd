# 凭证管理方案说明

## 概述

本项目采用**编译时嵌入**的方式管理百度网盘 API 凭证，参考 [pan-player](https://github.com/liuguanyu/pan-player) 的实现方式。

## 文件结构

```
internal/config/
├── credentials.go          # 实际凭证（不提交到 git）
├── credentials.go.example  # 示例模板（提交到 git）
└── config.go              # 配置加载逻辑
```

## 实现方式

### 1. 凭证定义 (credentials.go)

实际凭证以 Go 常量形式定义：

```go
package config

type BaiduCredentials struct {
    AppID        string
    ClientID     string
    ClientSecret string
    SignKey      string
    RedirectURI  string
    Scope        string
    DeviceName   string
}

// 内置的百度网盘 API 凭证
// 这些凭证会在编译时嵌入到程序中
var BaiduCredentialsConfig = BaiduCredentials{
    AppID:        "25658982",
    ClientID:     "pVB2TAdcOLZiCldLEcG1dABS3OK2owVi",
    ClientSecret: "XwXk28lcgoWLVlVLEkTMFxnqwA4onOLd",
    SignKey:      "rP4Btn+qap$stv3AstL+!XIpURM3mLn%",
    RedirectURI:  "oob",
    Scope:        "basic,netdisk",
    DeviceName:   "度盘读天下",
}
```

### 2. 示例模板 (credentials.go.example)

提供结构示例，提交到 git：

```go
package config

type BaiduCredentials struct {
    AppID        string
    ClientID     string
    ClientSecret string
    // ...
}

// 示例凭证配置（请替换为您自己的凭证）
var exampleCredentials = BaiduCredentials{
    AppID:        "YOUR_APP_ID",
    ClientID:     "YOUR_CLIENT_ID",
    ClientSecret: "YOUR_CLIENT_SECRET",
    // ...
}
```

### 3. Git 保护

`.gitignore` 配置：

```gitignore
# 百度网盘 API 凭证文件（不提交到版本控制）
internal/config/credentials.go
```

验证：

```bash
$ git check-ignore -v internal/config/credentials.go
.gitignore:43:internal/config/credentials.go  internal/config/credentials.go

$ git status --short internal/config/
A  internal/config/config.go
A  internal/config/credentials.go.example
# credentials.go 被忽略
```

## 使用流程

### 对于用户

1. 克隆项目
2. 编译运行（凭证已内置，无需配置）

```bash
git clone <repository>
cd pan-player-cmd
go build -o pan-player ./cmd/pan-player
./pan-player
```

### 对于开发者

如需使用自己的凭证：

1. 复制示例文件：
   ```bash
   cp internal/config/credentials.go.example internal/config/credentials.go
   ```

2. 编辑 `credentials.go`，填入实际凭证

3. 编译程序：
   ```bash
   go build -o pan-player ./cmd/pan-player
   ```

## 安全特性

### ✅ 编译时嵌入

- 凭证在编译时嵌入到二进制文件中
- 运行时不依赖外部配置文件
- 用户无法直接查看源码中的凭证

### ✅ Git 保护

- 实际凭证文件不提交到版本控制
- 只有示例模板被提交
- 即使克隆仓库也不会泄露凭证

### ✅ 参考 pan-player

本方案参考了 [pan-player](https://github.com/liuguanyu/pan-player) 的实现：

- pan-player: `src/config/credentials.ts` + `credentials.ts.example`
- 本项目: `internal/config/credentials.go` + `credentials.go.example`

## 优势对比

### vs. 运行时读取配置文件

| 方案 | 优点 | 缺点 |
|------|------|------|
| **编译时嵌入** | 无需外部配置文件<br>凭证不暴露给用户<br>部署简单 | 修改凭证需重新编译 |
| 运行时读取 | 可动态修改凭证 | 需要管理配置文件<br>明文存储凭证<br>用户可能误删配置 |

### vs. YAML 配置文件

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Go 代码常量** | Go 惯用方式<br>类型安全<br>编译时检查 | 非开发人员不易修改 |
| YAML 文件 | 人类可读<br>易于编辑 | 需要解析库<br>运行时加载 |

## 验证凭证嵌入

编译后验证凭证已嵌入：

```bash
$ strings pan-player | grep -o "pVB2TAdcOLZiCldLEcG1dABS3OK2owVi"
pVB2TAdcOLZiCldLEcG1dABS3OK2owVi
```

## 获取凭证

如需使用自己的百度网盘 API 凭证：

1. 访问 [百度网盘开放平台](https://pan.baidu.com/union/)
2. 创建应用
3. 获取 App Key (client_id) 和 Secret Key (client_secret)
4. 更新 `credentials.go` 并重新编译

## 总结

本方案实现了：
- ✅ 凭证内置在程序中，无需用户配置
- ✅ Git 保护实际凭证文件
- ✅ 提供示例模板供开发者参考
- ✅ 符合 Go 语言习惯
- ✅ 参考 pan-player 的成熟方案

用户可以直接编译运行，无需任何配置即可使用百度网盘 API 功能。
