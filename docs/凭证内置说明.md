# 百度网盘 API 凭证内置说明

## 变更说明

为了简化用户配置流程，百度网盘 API 的 `client_id` 和 `client_secret` 已内置到程序中，用户无需手动配置这些参数。

## 凭证来源

凭证从 [pan-player](https://github.com/liuguanyu/pan-player) 项目复制而来，仅供学习和个人使用。

## 文件位置

- **凭证定义**: `internal/config/credentials.go`
- **配置管理**: `internal/config/config.go`

### 2. 配置加载逻辑

在 `internal/config/config.go` 中：

```go
func LoadConfig() (*Config, error) {
    // 尝试读取配置文件
    data, err := os.ReadFile(configPath)

    // 如果配置文件不存在，使用默认配置（包含内置凭证）
    if err != nil {
        cfg := DefaultConfig()
        return cfg, nil
    }

    // 解析配置文件
    var cfg Config
    yaml.Unmarshal(data, &cfg)

    // 确保凭证始终使用内置值（覆盖用户配置）
    cfg.API.BaiduPan.ClientID = BaiduClientID
    cfg.API.BaiduPan.ClientSecret = BaiduClientSecret

    return &cfg, nil
}
```

## 用户影响

### 优点
- ✅ 无需手动配置 API 凭证
- ✅ 开箱即用，降低使用门槛
- ✅ 避免配置错误导致的登录失败

### 配置简化

**之前的配置方式**（已废弃）：
```yaml
api:
  baidu_pan:
    client_id: your_client_id        # 需要手动填入
    client_secret: your_client_secret # 需要手动填入
```

**现在的配置方式**：
- 用户无需关心 `client_id` 和 `client_secret`
- 只需运行程序，扫描二维码即可登录
- 可以自定义播放器、UI 等配置

### 为什么凭证可以内置？

1. **OAuth 2.0 设备码授权**：凭证用于获取设备码，用户需要主动扫码授权
2. **回调地址限制**：`redirect_uri=oob` 确保令牌只能由用户手动复制
3. **开源项目**：作为学习和个人使用项目，凭证暴露风险可控
4. **参考实现**：凭证来自开源项目 pan-player，已在社区公开使用

### 用户数据保护

用户登录后的 `token.json` 和其他数据保存在 `~/.pan-player/` 目录：

```
~/.pan-player/
├── config.yaml        # 配置文件（不含敏感凭证）
├── token.json         # 访问令牌（已加入 .gitignore）
├── playlists/         # 播放列表
└── cache/             # 音频缓存
```

## 登录流程

### 用户体验

1. 运行 `./pan-player`
2. 程序自动使用内置凭证获取设备码
3. 显示二维码和验证链接
4. 用户使用手机百度网盘 APP 扫码授权
5. 程序自动获取访问令牌并保存

### 技术实现

```go
// 1. 获取设备码
deviceAuth, err := api.GetDeviceCode(
    config.BaiduClientID,     // 使用内置凭证
    config.BaiduClientSecret,
)

// 2. 显示二维码
qrURL := fmt.Sprintf("%s?code=%s", deviceAuth.VerificationURL, deviceAuth.UserCode)
qrCode := generateQRCode(qrURL)

// 3. 轮询等待授权
tokenResp, err := api.WaitForAuth(ctx,
    config.BaiduClientID,
    config.BaiduClientSecret,
    deviceCode,
    interval,
)

// 4. 保存令牌
api.SaveToken(tokenInfo)
```

## 常见问题

### Q: 凭证会被滥用吗？
A: 凭证只用于 OAuth 设备码授权，用户需要主动扫码授权。即使凭证泄露，也无法获取用户的访问令牌。

### Q: 可以使用自己的凭证吗？
A: 可以，但不推荐。程序会自动覆盖用户配置文件中的凭证，确保使用内置凭证。

### Q: Token 会过期吗？
A: 会的，百度网盘 Token 有效期约为 30 天。过期后需要重新扫码登录。

### Q: 如何更换凭证？
A: 修改 `internal/config/credentials.go` 中的常量，然后重新编译程序。
